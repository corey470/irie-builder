# PSYCHOLOGY.md — Irie Behavioral Design Layer

> Read alongside DESIGN.md before building any UI. Design tokens tell you what it looks like. This tells you how it makes people feel.

## The North Star

Every screen, section, and interaction should answer one question:
**Does this make someone feel like everything is handled?**
That is Irie. Total peace. Nothing falling through the cracks.

---

## Core Psychological Principles

### 1. Trust Before Ask
Never ask for something before you've given something first.
- Show value before showing a form
- Show the product before showing the price
- Show the brand before showing the CTA
- Rule: Every page earns the right to ask for action

### 2. Momentum Over Friction
Every scroll, click, and interaction should feel like it's carrying the user forward — not stopping them.
- No dead ends. Every empty state has a next step.
- No walls. Every gate (login, payment) is preceded by desire.
- No confusion. One primary action per screen.
- Rule: If someone has to think about what to do next, we failed.

### 3. Specificity Builds Trust
Generic copy feels like a lie. Specific copy feels like truth.
- Bad: "Premium quality garments"
- Good: "Printed in-house in New Smyrna Beach. Small drops. Never restocked."
- Bad: "Respond faster to leads"
- Good: "Reply while you're still in the truck"
- Rule: Replace every adjective with a fact or a scene

### 4. Scarcity Without Manipulation
Create real urgency from real constraints — not fake countdown timers.
- Small drops that actually sell out
- In-house printing that actually takes time
- One operator who actually answers
- Rule: Only use urgency if it's true

### 5. Social Proof Placement
Put proof at the moment of doubt, not at the top of the page.
- Doubt happens at the CTA — put reviews/trust signals right before the button
- Doubt happens on the product page — put size/quality notes near Add to Cart
- Doubt happens at checkout — put guarantee/shipping info on the payment screen
- Rule: Proof goes where fear lives

### 6. The Emotion Sequence
Every page follows this emotional arc:
1. **Recognition** — "This is for me" (hero, first 3 seconds)
2. **Curiosity** — "Tell me more" (scroll trigger, first product/feature)
3. **Desire** — "I want this" (product detail, outcome copy)
4. **Trust** — "I believe this" (proof, specificity, brand story)
5. **Action** — "I'm doing this" (CTA, frictionless checkout)

Never skip steps. Never reorder them.

### 7. Voice = Personality = Trust
Copy written in Corey's actual voice builds more trust than polished marketing copy.
- Write like you're talking to someone at a stoplight
- Short sentences. Real words. No buzzwords.
- If it sounds like an ad, rewrite it
- Rule: Would Corey actually say this? If no, cut it.

## Page-Level Psychology

### Homepage / Landing
- Hero: Make them feel something in 3 seconds. No explaining, just feeling.
- Scroll 1: Show the thing they came for (product, service, result)
- Scroll 2: Make them believe it (specificity, story, proof)
- Scroll 3: Make it easy to act (clear CTA, no friction)

### Product / Service Page
- Lead with the outcome, not the features
- Show the real thing — real photos, real copy, real price
- Put the CTA above the fold AND after the proof
- Never hide the price

### Checkout / Conversion
- Remove everything that isn't the action
- Restate what they're getting (not what they're paying)
- One thing on the screen at a time
- Show trust signals (guarantee, shipping, security) at the moment of hesitation

### Empty States
- Every empty state is an opportunity, not a failure
- Never show a blank screen — always show a path forward
- Copy: "Nothing here yet" + specific next action
- Never: Just "No results"

### Error States
- Own the error — don't blame the user
- Tell them exactly what happened
- Tell them exactly what to do next
- Never: "Something went wrong"

## Motion & Scroll Psychology

### Scroll reveals = dopamine hits
- Reveal content progressively — don't show everything at once
- Each scroll should feel like unwrapping something
- Use scroll-triggered animations to reward attention

### Timing
- Entrance animations: 300–500ms (fast enough to feel snappy)
- Content reveals: 400–600ms with slight delay stagger between elements
- Hover states: 150–200ms (instant-feeling)
- Page transitions: 200–300ms

### What motion should say
- "This is alive" — not "look how animated this is"
- Motion should reinforce meaning, not decorate
- Rule: If removing the animation changes nothing about the message, remove the animation

## Copywriting Rules

### Headlines
- Lead with the feeling, not the feature
- Max 8 words for hero headlines
- Use present tense ("You feel") not future ("You will feel")

### CTAs
- Say what happens next, not what the button does
- Bad: "Submit" / "Click Here" / "Learn More"
- Good: "Get My Spot" / "Start Building" / "Shop the Drop"
- One CTA per screen. Two CTAs = no CTA.

### Body Copy
- Front-load the most important word in every sentence
- Cut every sentence by 30% after you write it
- Read it out loud. If you stumble, rewrite it.

## The Irie Test
Before shipping any screen, ask:
1. Does this make someone feel like everything is handled?
2. Does this feel warm, confident, and real — not corporate?
3. Would Corey say this to a customer face to face?
4. Is there any friction we haven't eliminated?
5. Does every empty state have a next step?

If any answer is no — rework it before shipping.
